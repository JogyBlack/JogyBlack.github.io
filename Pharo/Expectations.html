What to expect of Borland's free C/C++ commandline tools

The free commandline tools are:

  • commandline tools only: To invoke the tools you need to type their name and a sequence of arguments on the command line of a Command Prompt. On most Win32 systems you can start such a prompt via "Start|Programs|Command Prompt". There is no graphical development environment included with the free tools download. Not even a source code editor!
  • for developping Win32 applications only: You can use this tools to create .EXE and .DLL files to be run in Win32 environments. There is no way to use the free commandline tools to create applications targeted for DOS or Win16. If you need to create programs for DOS then you may download an ancient version of Turbo C at Borland's Museum (you need to become a member first) 
  • to be run in a Win32 environment only: Even though most tools have no graphical user interface they are true Win32 applications and you can not use them under DOS or Win16
  • not enabled to build C++ Builder components (no VCL)
  • not enabled to assemble applications from C++ Builder components. (no VCL)
  • not instantly ready to do COM/OLE/ACTIVEX stuff. This can be partially solved in combination with a copy of Microsofts Platform SDK especially the Tools.
  • for making programs using the implementation languages C and C++ according to the latest standards. And last but not least they are
  • for free!
