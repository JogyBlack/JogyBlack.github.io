Setting up the free commandline tools

HOWTO

The download will leave you with a big executable file. Double click it in the Explorer window. When asked where you want the free tools to be installed make sure you select a valid path (e.g. do not use e:\c++). Eventually lots of files are extracted to your hard disk. Afterwards read the README.TXT file contained in the folder you have specified during setup. Follow the instructions in the README e.g. create the files bcc32.cfg and ilink32.cfg.

Hints

  • Place bcc32.cfg and ilink32.cfg in the BIN subdirectory of your installation. For example: If your installation is in C:\Borland\Bcc55 then these files must be placed in C:\Borland\Bcc55\BIN. File bcc32.cfg will set the compiler options for the Include and Lib paths (compiler options -I and -L). Add these lines:
      
    -I"C:\Borland\Bcc55\include"
-L"C:\Borland\Bcc55\lib;C:\Borland\Bcc55\lib\PSDK"

    File ilink32.cfg will set the linker option for the Lib path (linker option -L). Add this line:
      
    -L"C:\Borland\Bcc55\lib;C:\Borland\Bcc55\lib\PSDK"

    Note: If your installation is not in C:\Borland\Bcc55 then you have to adapt the content of bcc32.cfg and ilink32.cfg to contain the path strings of your installation.

  • If you use Notepad for the task of creating bcc32.cfg and ilink32.cfg make sure to put double-qoutes around the filenames in the save-as dialog. Otherwise you will be creating bcc32.cfg.txt and ilink32.cfg.txt which will be ignored by the tools.

  • Bcc32 should be on your PATH somewhere. For example: If your installation is in C:\Borland\Bcc55 then you should add C:\Borland\Bcc55\Bin to the PATH environment variable of your system. On Win9x this can be done by editing the file Autoexec.bat (in the root directory of the systems boot drive) and appending this line:
      
    SET PATH=%PATH%;C:\Borland\Bcc55\Bin

    The new PATH setting will not have effect until you reboot your system.
    Note: If the PATH already contains a directory with some Borland tools in it - originating from an older installation - then those tools will be used! You can force the system to use the newly installed tools by setting the PATH with SET PATH=C:\Borland\Bcc55\Bin;%PATH% but this might affect the older installation in a bad way. One solution is to modify the PATH for each command prompt only instead of modifying the systems settings:
     
    Got another BORLAND product installed?/Can't change system's PATH?
    If for some reason the system's PATH can/should not be changed you might follow these instructions: Create a batch file named SETBCC55ENV.BAT with content:
      
    SET PATH=C:\Borland\Bcc55\Bin;%PATH%

    If possible place SETBCC55ENV.BAT in a folder which is already on the PATH. For each command prompt invoke SETBCC55ENV before invoking any of the free commandline tools.

  • There is no graphical environment in the free commandline tools installation. To use the tools open a command prompt (Start|Programs|Command Prompt) and issue commands there. For source code editing and makefile generation consider using VIDE.

  • For a survey on the commandline syntax:
    1. Open a command prompt
    2. Type bcc32 | more followed by a return. See an explanation of the compiler syntax and options - at least if bcc32.exe can be found on the PATH.
    3. Type ilink32 | more followed by a return. This will show the linker syntax and an explanation of some linker options.

    For more information on the commandline usage/options refer to BCB5TOOL.HLP which can be found in your installations HELP subfolder. Place a shortcut to this help file in some convenient place, e.g. in the Start menu or on the desktop. Additionally some documentation on the C/C++ language and it's library can be downloaded from Borland's developper support or Borland's techpubs site

  • Compatibility with pre Win2000 systems: If you intend to deploy your programs on systems running NT4, Win95 or Win98 you should append these lines at the end of your bcc32.cfg file:
      
    -DWINVER=0x0400
-D_WIN32_WINNT=0x0400
